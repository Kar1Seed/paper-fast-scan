摘要 Abstract

We present QLoRA, an efficient finetuning approach that reduces memory usage enough to finetune a 65B parameter model on a single 48GB GPU while preserving full 16-bit finetuning task performance. QLoRA backpropagates gradients through a frozen, 4-bit quantized pretrained language model into Low Rank Adapters (LoRA). Our best model family, Guanaco, outperforms all previous openly released models on the Vicuna benchmark, reaching 99.3% of ChatGPT performance with only 24 hours of finetuning. Three innovations: 4-bit NormalFloat (NF4) — information-theoretically optimal for normally distributed weights; Double Quantization — quantizes quantization constants, saving 0.37 bits/parameter (~3GB for 65B); Paged Optimizers — using NVIDIA unified memory to handle memory spikes. We train 1,000+ models across 8 datasets, showing that data quality trumps dataset size (9k OASST1 > 450k FLAN v2) and that MMLU performance does not imply chatbot performance.
我们提出 QLoRA，将内存压缩到单张 48GB GPU 微调 65B 模型，同时保持全 16-bit 微调性能。QLoRA 通过冻结的 4-bit 量化模型将梯度反向传播到 LoRA 低秩适配器。最佳模型 Guanaco 在 Vicuna 基准上超越所有开源模型，仅 24 小时微调达 ChatGPT 性能的 99.3%。三项创新：NF4 数据类型——正态分布的信息论最优量化；双重量化——量化量化常数，每参数省 0.37 bit（65B 省 ~3GB）；分页优化器——利用 NVIDIA 统一内存处理显存尖峰。训练 1000+ 模型后证明：数据质量碾压数量（9k OASST1 > 450k FLAN v2），且 MMLU 高分不代表聊天能力强。

1. 引言 Introduction

Finetuning LLMs is highly effective but prohibitively expensive — regular 16-bit finetuning of LLaMA 65B requires over 780 GB of GPU memory. While quantization methods reduce inference memory, they break down during training. QLoRA demonstrates for the first time that a quantized 4-bit model can be finetuned without performance degradation. Key findings from training 1,000+ models: (1) 9k high-quality samples (OASST1) outperform 450k samples (FLAN v2) on chatbot benchmarks; (2) MMLU performance does NOT predict chatbot quality; (3) GPT-4 evaluation largely agrees with human evaluation, providing a cheap alternative; (4) Current chatbot benchmarks are not trustworthy for accurate performance evaluation.
LLM 微调效果显著但成本极高——LLaMA 65B 的 16-bit 微调需超 780 GB GPU 显存。量化方法虽降低推理内存，但训练时失效。QLoRA 首次证明4-bit 量化模型可无损微调。1000+ 模型训练揭示：(1) 9k 高质量样本 (OASST1) 聊天性能碾压 450k 样本 (FLAN v2)；(2) MMLU 分数不能预测聊天质量；(3) GPT-4 评估与人类评估高度一致，提供廉价替代；(4) 当前聊天基准不可信，无法准确评估性能。

2. 背景 Background

2.1 分块 k-bit 量化

Quantization discretizes data from higher to lower bit representations. Standard approach: rescale input to target range via absolute maximum normalization: $X_{\text{Int8}} = \text{round}\!\left(\frac{127}{\text{absmax}(X_{\text{FP32}})} X_{\text{FP32}}\right) = \text{round}(c \cdot X)$. Dequantization: $\text{dequant}(c, X_{\text{Int8}}) = X_{\text{Int8}} / c$. The problem: outliers cause under-utilization of quantization bins. Solution: block-wise quantization — chunk the tensor into $n$ contiguous blocks of size $B$, each independently quantized with its own constant $c_i$. This prevents a single outlier from ruining the entire tensor's precision.
量化将数据从高位宽离散化到低位宽。标准方法：通过绝对最大值归一化缩放到目标范围。问题：离群值导致量化桶利用不足。解法：分块量化——将张量切为 $n$ 个大小为 $B$ 的连续块，每块独立量化。避免单个离群值破坏整个张量精度。
$$X_{\text{Int8}} = \text{round}\!\left(\frac{127}{\text{absmax}(X_{\text{FP32}})} \cdot X_{\text{FP32}}\right) = \text{round}(c \cdot X_{\text{FP32}})$$

2.2 LoRA 训练内存分析

LoRA augments a linear projection $Y = XW$ with a low-rank factorized projection: $Y = XW + s \cdot X L_1 L_2$, where $L_1 \in \mathbb{R}^{h \times r}$, $L_2 \in \mathbb{R}^{r \times o}$. Crucially, most memory comes from activation gradients, not LoRA parameters. For LLaMA 7B: LoRA parameters = 26 MB, input gradients = 567 MB (reduced to 18 MB with gradient checkpointing), 4-bit base model = 5,048 MB. This means we can use more adapters without significantly increasing memory — key for recovering full 16-bit performance.
LoRA 用低秩分解增强线性投影。关键：大部分内存来自激活梯度，不是 LoRA 参数。LLaMA 7B：LoRA 参数 = 26 MB，输入梯度 = 567 MB（梯度检查点后降至 18 MB），4-bit 基座模型 = 5,048 MB。这意味着可以加更多适配器而不显著增加内存——这是恢复全精度性能的关键。
$$Y = XW + s \cdot X L_1 L_2,\quad L_1 \in \mathbb{R}^{h \times r},\; L_2 \in \mathbb{R}^{r \times o}$$

3. QLoRA 方法

3.1 4-bit NormalFloat (NF4)

NormalFloat builds on Quantile Quantization — each bin gets an equal number of values, information-theoretically optimal. Since pretrained NN weights are zero-centered normal with std σ, we can precompute exact quantiles for $\mathcal{N}(0,1)$ once and scale. For k-bit NFk, estimate $2^k$ quantiles $q_i$: $q_i = \frac{1}{2}\!\left(Q_X\!\left(\frac{i}{2^k+1}\right) + Q_X\!\left(\frac{i+1}{2^k+1}\right)\right)$, where $Q_X$ is the standard normal quantile function. To ensure an exact zero representation, NF4 uses an asymmetric design: $2^{k-1}$ negative quantiles + $2^{k-1}+1$ positive quantiles (removing duplicate zero). NF4 empirically outperforms 4-bit Integer and 4-bit Float.
NormalFloat 基于分位数量化——每桶等量赋值，信息论最优。由于预训练权重是零中心正态分布，可预计算 $\mathcal{N}(0,1)$ 的精确分位数后缩放。为确保精确表示零，NF4 采用非对称设计：$2^{k-1}$ 负分位数 + $2^{k-1}+1$ 正分位数。NF4 实验优于 4-bit Integer 和 4-bit Float。
$$q_i = \frac{1}{2}\!\left(Q_X\!\left(\frac{i}{2^k+1}\right) + Q_X\!\left(\frac{i+1}{2^k+1}\right)\right),\quad Q_X = \Phi^{-1}(\cdot)$$

3.2 双重量化 Double Quantization

Block-wise 4-bit quantization uses 32-bit constants per block. With blocksize 64, this adds 0.5 bits/parameter of overhead. Double Quantization applies a second quantization to the constants themselves: $c^{\text{FP32}}_2 \xrightarrow{\text{quantize}} c^{\text{FP8}}_2$ with blocksize 256, producing second-level constants $c^{\text{FP32}}_1$. Since $c_2$ values are positive, we subtract the mean before quantizing for symmetric use. Result: overhead drops from 0.5 → 0.127 bits/parameter (saving 0.373 bits/param, ~3GB for 65B model). No performance degradation observed for 8-bit second quantization.
分块 4-bit 量化每块使用32-bit 常数。blocksize 64 时，开销 = 0.5 bit/参数。双重量化对常数本身进行第二次量化：blocksize 256 的 8-bit 量化。由于 $c_2$ 为正，减均值后对称量化。结果：开销从 0.5 → 0.127 bit/参数（每参数省 0.373 bit，65B 模型省 ~3GB）。8-bit 第二级量化无性能损失。
$$\text{Overhead reduction: } \frac{32}{64} = 0.5 \;\rightarrow\; \frac{8}{64} + \frac{32}{64 \cdot 256} = 0.127 \text{ bits/param}$$

3.3 分页优化器 Paged Optimizers

Gradient checkpointing causes memory spikes when processing mini-batches with long sequences — a frequent OOM scenario. Paged Optimizers leverage NVIDIA unified memory, which automatically pages data between CPU RAM and GPU memory. Optimizer states are allocated in paged memory; when GPU runs out, states are evicted to CPU RAM, then paged back when needed during the optimizer update step. This prevents OOM errors without code changes to the training loop.
梯度检查点在处理长序列 mini-batch 时导致显存尖峰——常见的 OOM 来源。分页优化器利用 NVIDIA 统一内存，CPU 和 GPU 间自动分页。优化器状态分配在分页内存中；GPU 显存不足时驱逐到 CPU RAM，优化器更新时按需分页回 GPU。无需修改训练循环代码即可防止 OOM。

3.4 QLoRA 完整定义

QLoRA uses one storage type (NF4) and one computation type (BF16). For a single linear layer with LoRA adapter:
QLoRA 使用一种存储类型 (NF4) 和一种计算类型 (BF16)。对带 LoRA 的单个线性层：
$$Y^{\text{BF16}} = X^{\text{BF16}} \cdot \text{doubleDequant}(c_1^{\text{FP32}}, c_2^{\text{k-bit}}, W^{\text{NF4}}) + X^{\text{BF16}} L_1^{\text{BF16}} L_2^{\text{BF16}}$$ $$\text{doubleDequant}(c_1, c_2, W^{\text{k-bit}}) = \text{dequant}(\text{dequant}(c_1, c_2), W^{\text{4-bit}}) = W^{\text{BF16}}$$
Only LoRA adapter gradients $\partial E/\partial L_i$ are computed; 4-bit weights $\partial E/\partial W$ are never needed. Forward/backward passes dequantize on-the-fly, but gradients only flow through the 16-bit LoRA parameters.
仅计算LoRA 适配器梯度 $\partial E/\partial L_i$；无需 4-bit 权重的梯度。前向/反向传播即时反量化，但梯度仅流经 16-bit LoRA 参数。

4. 实验：QLoRA vs 标准微调

4.1 GLUE 与指令微调

Evaluated on three architectures (encoder RoBERTa, encoder-decoder T5, decoder-only LLaMA) against 16-bit full finetuning and 16-bit LoRA. Results: QLoRA matches 16-bit performance within ±1% across GLUE (RoBERTa-large), Super-NaturalInstructions (T5), and MMLU (LLaMA 7B-65B). NF4 consistently outperforms Float4 (+0.3-0.5% on average). Double Quantization shows no measurable accuracy loss. With LoRA on all linear layers, QLoRA achieves full parity with 16-bit finetuning — critical for practical adoption.
三种架构与全 16-bit 微调和16-bit LoRA对比。结果：GLUE、Super-NI、MMLU 上 QLoRA 与 16-bit 差距 ±1% 以内。NF4 持续优于 Float4（平均 +0.3-0.5%）。双重量化无显著精度损失。所有线性层加 LoRA 后，QLoRA 实现与 16-bit 完全持平——对实际采用至关重要。

4.2 消融实验

NF4 vs Float4: NF4 consistently better (+0.3-0.5%). NF4's theoretical optimality for normal distributions translates to empirical gains. Adapter placement: applying LoRA to all linear layers (attention + MLP) is crucial — attention-only LoRA recovers only ~95% of full performance. Rank r: r=64 gives best results for 65B, while r=16 suffices for 7B. Larger models benefit from higher rank adapters. Blocksize: B=64 for W gives best precision/memory trade-off. B=256 for $c_2$ (Double Quant) is optimal.
NF4 vs Float4：NF4 持续更优。正态分布的理论最优性转化为实际收益。适配器位置：所有线性层加 LoRA 至关重要——仅注意力层只能恢复 ~95% 性能。秩 r：65B 用 r=64 最佳，7B 用 r=16 即可。大模型受益于更高秩适配器。块大小：W 用 B=64 精度/内存最优，$c_2$ 用 B=256（双重量化）最佳。

5. Guanaco 聊天机器人

5.1 Elo 评分体系

Tournament-style evaluation: models compete head-to-head, judged by GPT-4 or human annotators. Winners aggregated into Elo ratings over 10,000 random orderings. Key results: Guanaco 65B reaches Elo 1022±1, surpassing ChatGPT (966±1) and Vicuna 13B (974±1). Guanaco 33B (Elo 992) also beats ChatGPT. Guanaco 13B (Elo 916, only 10 GB memory) surpasses Bard (902). Smallest Guanaco 7B (6 GB) outperforms 26 GB Alpaca by 20+ percentage points.
锦标赛评估：模型两两对战，由GPT-4或人类标注裁判。胜者汇聚为Elo 评分（10,000 次随机排序）。结果：Guanaco 65B Elo 1022±1，超越 ChatGPT (966) 和 Vicuna 13B (974)。Guanaco 33B (Elo 992) 也击败 ChatGPT。Guanaco 13B (仅 10 GB 显存) 超越 Bard (902)。最小 Guanaco 7B (6 GB) 比 26 GB Alpaca 高20+ 百分点。

5.2 数据质量 vs 数量

One of the paper's most impactful findings: 9k high-quality OASST1 samples produce better chatbots than 450k FLAN v2 samples. Quality > Quantity. MMLU score does not correlate with chatbot Elo — a model can score high on MMLU but be a terrible conversationalist. GPT-4 as evaluator: GPT-4 judgments agree with humans ~80% of the time, with strong disagreement in ~5% of cases. Model-based evaluation is a cheap, reasonable alternative but has uncertainties — both GPT-4 and human evaluation have biases.
论文最具影响力的发现：9k OASST1 高质量样本胜于 450k FLAN v2 样本。质量 > 数量。MMLU 分数与聊天 Elo 不相关——模型可以 MMLU 高分但聊天很差。GPT-4 做裁判：GPT-4 判断与人类约 80% 一致，~5% 案例强烈分歧。基于模型的评估是廉价合理的替代，但有不确定性——GPT-4 和人类评估都有偏见。

6. 定性分析

Guanaco shows strong performance but fails in specific areas: factual accuracy (hallucination on obscure facts), mathematical reasoning (incorrect step-by-step solutions), and over-refusal (declining innocuous requests judged as potentially harmful). These failures are not captured by quantitative benchmarks, highlighting their inadequacy. The team open-sourced all model generations with GPT-4 and human annotations, along with 32 fine-tuned adapter sets (7/13/33/65B × 8 datasets) and CUDA kernels for 4-bit training.
Guanaco 表现强劲但在特定领域失败：事实准确度（冷门知识幻觉）、数学推理（步骤错误）、过度拒绝（误判无害请求）。这些失败不被量化基准捕捉，凸显其不足。团队开源全部模型生成及 GPT-4/人类标注，以及 32 套微调适配器（7/13/33/65B × 8 数据集）和 4-bit 训练 CUDA 内核。

7. 局限 Limitations

QLoRA's limitations: (1) NF4 is optimal only for normal distributions — other distributions (e.g., after fine-tuning) may benefit from different data types; (2) benchmarks are unreliable — neither MMLU nor Vicuna fully captures model quality; (3) training is still slower than full finetuning per step (dequantization overhead), though the memory savings enable training otherwise-impossible models; (4) GPT-4 evaluation has biases — it prefers longer, more verbose responses and has positional bias (preferring first response). Future work should develop better benchmarks and faster 4-bit matrix multiplication kernels.
QLoRA 局限：(1) NF4 仅对正态分布最优——其他分布可能需不同数据类型；(2) 基准不可靠——MMLU 和 Vicuna 都不能完全反映模型质量；(3) 训练每步比全微调慢（反量化开销），但内存节省使能训练原本不可能的模型；(4) GPT-4 评估有偏见——偏好更长回复、有位置偏向（偏好第一个回复）。未来应开发更好基准和更快 4-bit 矩阵乘法内核。
