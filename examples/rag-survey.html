摘要 Abstract

Large Language Models (LLMs) showcase impressive capabilities but encounter challenges like hallucination, outdated knowledge, and non-transparent, untraceable reasoning. Retrieval-Augmented Generation (RAG) has emerged as a promising solution by incorporating knowledge from external databases. This enhances accuracy and credibility, particularly for knowledge-intensive tasks, and allows for continuous knowledge updates and integration of domain-specific information. This survey examines the progression of RAG paradigms: Naive RAG, Advanced RAG, and Modular RAG. It scrutinizes the tripartite foundation: retrieval, generation, and augmentation techniques, along with evaluation frameworks, benchmarks, challenges, and future directions.
大语言模型 (LLM) 展现出令人瞩目的能力，但面临幻觉、知识过时和推理不透明不可追溯等挑战。检索增强生成 (RAG) 通过引入外部知识库中的信息，显著提升了准确性和可信度，尤其适用于知识密集型任务，并支持知识的持续更新和领域信息的整合。本综述系统梳理了 RAG 范式的演进：朴素 RAG、高级 RAG 和模块化 RAG。深入剖析了三大核心支柱——检索、生成、增强的技术细节，并涵盖了评估框架、基准测试、当前挑战与未来方向。

I. 引言 Introduction

LLMs face significant limitations in domain-specific and knowledge-intensive tasks, notably producing hallucinations when handling queries beyond their training data. RAG addresses this by retrieving relevant document chunks from external knowledge bases through semantic similarity calculation. The development trajectory shows three stages: early work focused on pre-training enhancement with Transformers; the ChatGPT era shifted to inference-stage RAG leveraging in-context learning; recent work integrates RAG with LLM fine-tuning. This survey analyzes over 100 RAG studies across three paradigms.
LLM 在领域特定和知识密集型任务中存在显著局限，特别是面对训练数据之外的问题时会产生幻觉。RAG 通过语义相似度计算从外部知识库检索相关文档片段来解决此问题。发展轨迹分三个阶段：早期聚焦于 Transformer 架构的预训练增强；ChatGPT 时代转向利用上下文学习的推理阶段 RAG；近期研究则将 RAG 与LLM 微调深度融合。本综述分析了 100 余篇 RAG 研究。
Fig 1: RAG 技术树。RAG 研究阶段包括预训练、微调和推理。随着 LLM 的兴起，RAG 研究最初聚焦于利用 LLM 强大的上下文学习能力（推理阶段），后续逐步与 LLM 微调技术深度融合。

II. RAG 概览 Overview of RAG

The paper categorizes RAG into three paradigms:

1. Naive RAG: The classic "Retrieve-Read" pipeline. Indexing → clean & chunk documents → encode into vectors → store in vector database. Retrieval → encode query → compute similarity → retrieve Top-K chunks. Generation → feed query + retrieved chunks into LLM → generate answer.

2. Advanced RAG: Enhances Naive RAG with pre-retrieval (query rewriting, routing) and post-retrieval (re-ranking, context compression) optimizations.

3. Modular RAG: Breaks the linear pipeline into interchangeable modules — search, memory, fusion, routing, task adaptation — enabling flexible composition for different scenarios.
论文将 RAG 分为三种范式：

1. 朴素 RAG: 经典的"检索-阅读"流水线。索引→清洗并分块文档→向量编码→存入向量数据库。检索→编码查询→计算相似度→取 Top-K 片段。生成→将查询+检索片段输入 LLM→生成答案。

2. 高级 RAG: 在朴素 RAG 上增加检索前（查询重写、路由）和检索后（重排序、上下文压缩）优化。

3. 模块化 RAG: 打破线性流程，变成可替换模块——搜索、记忆、融合、路由、任务适配——可灵活组合适配不同场景。
Fig 2: RAG 问答流程示例。三步：(1) 索引 — 文档分块→向量编码→向量数据库；(2) 检索 — 查询编码→语义相似度→Top-K 相关块；(3) 生成 — 查询+检索块→LLM→最终答案。

III. 检索 Retrieval

Retrieval optimization spans three dimensions:

Indexing Optimization: Chunking strategies (fixed-size, semantic, recursive), metadata attachment (timestamp, source, category for filtering), and multi-representation indexing (dense + sparse vectors, hybrid retrieval).

Query Optimization: Query rewriting (using LLM to rephrase ambiguous queries), query decomposition (breaking complex queries into sub-queries), HyDE (generating hypothetical documents to bridge semantic gap), and multi-query retrieval.

Embedding Optimization: Fine-tuning embedding models on domain-specific data, task-aware embeddings, and using LLMs themselves as embedding generators (e.g., LLM-Embedder).
检索优化覆盖三个维度：

索引优化: 分块策略（固定长度、语义分块、递归分块）、元数据附着（时间戳、来源、类别用于过滤）、多表示索引（稠密向量+稀疏向量，混合检索）。

查询优化: 查询重写（用 LLM 改写模糊查询）、查询分解（将复杂查询拆分为子查询）、HyDE（生成假设文档来桥接语义鸿沟）、多查询检索。

嵌入优化: 在领域数据上微调嵌入模型、任务感知嵌入、使用 LLM 本身作为嵌入生成器（如 LLM-Embedder）。
Fig 3: 三种 RAG 范式对比。Naive RAG（线性流水线）→ Advanced RAG（增加检索前后优化）→ Modular RAG（模块化可替换组件，支持搜索、记忆、融合、路由等多模块灵活编排）。

IV. 生成 Generation

Post-retrieval processing and generation enhancements:

Post-Retrieval Processing: Re-ranking (re-scoring retrieved chunks for relevance), Context compression (summarizing/compressing retrieved content to fit context window), and Context curation (selecting and ordering chunks for optimal prompt construction).

LLM Fine-tuning for RAG: Fine-tuning with retrieved context (training LLMs to better utilize retrieved information), instruction tuning for RAG-specific behaviors, and RLHF to align responses with retrieved evidence.

Controllable Generation: Mechanisms to ensure the LLM adheres to retrieved content, including citation generation (attributing claims to sources) and faithfulness constraints (preventing fabrication).
检索后处理与生成增强：

检索后处理: 重排序（对检索块重新评分）、上下文压缩（压缩检索内容以适应上下文窗口）、上下文编排（选择排序块以构建最优提示词）。

面向 RAG 的 LLM 微调: 带检索上下文的微调（训练 LLM 更好利用检索信息）、指令微调 RAG 专属行为、RLHF 对齐回答与检索证据。

可控生成: 确保 LLM 遵循检索内容的机制，包括引文生成（将声明归于来源）和忠实性约束（防止捏造）。

V. 增强 Augmentation

Three augmentation processes enhance RAG at different stages:

1. Pre-training Augmentation: RETRO (Retrieval-Enhanced Transformer) integrates retrieval directly into pre-training, building a model that natively understands how to use retrieved information. Atlas jointly trains retriever and generator.

2. Fine-tuning Augmentation: Augmenting the fine-tuning data with retrieved context to teach the model to leverage external knowledge. RA-DIT (Retrieval-Augmented Dual Instruction Tuning) is a representative approach.

3. Inference Augmentation: The most common form — dynamically retrieving relevant information during inference. Key techniques include iterative retrieval (multi-hop reasoning), recursive retrieval (retrieval → generation → retrieval loop), and adaptive retrieval (deciding when retrieval is needed via model confidence).
三个增强过程在不同阶段强化 RAG：

1. 预训练增强: RETRO（检索增强 Transformer）将检索直接集成到预训练中，构建原生理解如何使用检索信息的模型。Atlas 联合训练检索器和生成器。

2. 微调增强: 用检索上下文增强微调数据，教会模型利用外部知识。RA-DIT（检索增强双指令微调）是代表性方法。

3. 推理增强: 最常见的形式——推理时动态检索。关键技术包括迭代检索（多跳推理）、递归检索（检索→生成→检索循环）和自适应检索（通过模型置信度判断何时需要检索）。
Fig 4: RAG 增强过程分类。预训练增强（RETRO/Atlas）、微调增强（RA-DIT）、推理增强（迭代/递归/自适应检索），覆盖模型生命周期的三个阶段。

VI. 评估 Evaluation

The paper summarizes RAG evaluation across 26 tasks and nearly 50 datasets:

Downstream Tasks: Open-domain QA (Natural Questions, TriviaQA), Multi-hop reasoning (HotpotQA, 2WikiMultihopQA), Fact verification (FEVER), Dialogue, Summarization, Code generation.

Evaluation Metrics: Faithfulness (does the answer match retrieved evidence?), Answer Relevance, Context Relevance, Noise Robustness (handling irrelevant retrieved documents).

Benchmarks & Tools: RAGAS, ARES, TruLens for automated RAG evaluation; RGB, RECALL benchmarks.
论文总结了覆盖 26 个任务和近 50 个数据集的 RAG 评估体系：

下游任务: 开放域问答（Natural Questions, TriviaQA）、多跳推理（HotpotQA, 2WikiMultihopQA）、事实验证（FEVER）、对话、摘要、代码生成。

评估指标: 忠实性（答案是否匹配检索证据？）、答案相关性、上下文相关性、噪声鲁棒性（处理无关检索文档的能力）。

基准与工具: RAGAS、ARES、TruLens 用于自动化 RAG 评估；RGB、RECALL 基准测试。

VII. 挑战与未来 Challenges & Future

Key challenges and future directions:

Long Context & Multi-hop: How to effectively handle long documents and multi-step reasoning chains. Emerging solutions: iterative RAG, graph-based RAG.

Multi-modal RAG: Extending retrieval to images, tables, code — not just text. MM-RAG approaches integrate vision encoders.

Robustness & Trust: Handling noisy retrieval, adversarial attacks on retrievers, and ensuring the system knows when to say "I don't know."

RAG Ecosystem: The need for standardized evaluation, RAG-as-a-Service, and integration with AI Agents for autonomous knowledge workflows.
关键挑战与未来方向：

长上下文与多跳推理: 如何有效处理长文档和多步推理链。新兴方案：迭代 RAG、图基 RAG。

多模态 RAG: 将检索扩展到图像、表格、代码——不只文本。MM-RAG 方法整合视觉编码器。

鲁棒性与信任: 处理噪声检索、针对检索器的对抗攻击、确保系统知道何时说"我不知道"。

RAG 生态: 需要标准化评估、RAG 即服务、与AI Agent 集成实现自主知识工作流。

VIII. 结论 Conclusion

RAG represents a fundamental paradigm for bridging parametric knowledge (stored in model weights) with non-parametric knowledge (external databases). The evolution from Naive → Advanced → Modular RAG reflects a trend toward flexible, composable architectures. Key takeaways: (1) Retrieval quality is the foundation — poor retrieval poisons generation; (2) Augmentation timing matters — pre-training, fine-tuning, and inference each offer different benefits; (3) Evaluation is evolving — from simple QA accuracy to multi-dimensional metrics (faithfulness, relevance, robustness). RAG is not just a technique but an architectural paradigm that will underpin the next generation of trustworthy AI systems.
RAG 代表了连接参数化知识（存储于模型权重）与非参数化知识（外部数据库）的基础范式。从朴素→高级→模块化 RAG 的演进反映了向灵活、可组合架构发展的趋势。核心启示：(1) 检索质量是基础——糟糕的检索毒化生成；(2) 增强时机很重要——预训练、微调和推理各提供不同优势；(3) 评估体系在进化——从简单问答准确率到多维度指标（忠实性、相关性、鲁棒性）。RAG 不仅是一种技术，更是支撑下一代可信 AI 系统的架构范式。

📖 术语表 Glossary

RAG (检索增强生成)
在 LLM 生成答案前，先从外部知识库检索相关文档片段，将检索结果与原始查询一起输入模型。核心三阶段：索引→检索→生成。
Hallucination (幻觉)
LLM 生成看似合理但事实错误的内容。RAG 通过引入外部知识有效减少幻觉。
Naive RAG (朴素 RAG)
最基础的"检索-阅读"流水线：分块→向量化→相似度检索→Top-K→LLM 生成。简单但容易检索到不相关内容。
Advanced RAG (高级 RAG)
在朴素 RAG 上增加检索前（查询重写/路由）和检索后（重排序/上下文压缩）优化。
Modular RAG (模块化 RAG)
将 RAG 流程拆分为可替换的功能模块（搜索、记忆、融合、路由等），可灵活编排适配不同场景。
Vector Database (向量数据库)
存储文档块的向量表示，支持高效的语义相似度搜索。RAG 检索的核心基础设施。
Chunking (分块)
将长文档切分为小块以适应 LLM 上下文窗口。策略：固定长度、语义分块、递归分块。直接影响检索质量。
Re-ranking (重排序)
对初步检索结果用更精确的模型重新评分排序，提升 Top-K 片段的相关性。
HyDE
Hypothetical Document Embeddings：先生成假设答案文档→用假设文档的向量去检索，桥接查询与文档之间的语义鸿沟。
RETRO
Retrieval-Enhanced Transformer：DeepMind 提出的将检索直接整合进预训练的模型架构。
RA-DIT
Retrieval-Augmented Dual Instruction Tuning：Meta 提出的双指令微调方法，同时优化检索器和生成器。
Iterative RAG (迭代 RAG)
多轮检索→生成的循环过程，适用于多跳推理场景（需要多个检索步骤才能回答的问题）。
RAGAS
自动化 RAG 评估框架，评测忠实性、答案相关性、上下文相关性等指标。

🧮 RAG 核心流程

1. Indexing: Docs → Chunk → Embed → VectorDB
2. Retrieval: Query → Embed → Similarity → Top-K
3. Generation: (Query + Chunks) → LLM → Answer
